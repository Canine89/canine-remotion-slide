# Title Bullets Slide

## 언제 쓰나

- 개념 설명, 요점 정리, 핵심 주장 전개에 가장 기본으로 쓰는 타입
- 이미지 없이 메시지 자체를 전면에 세울 때 적합

## 타입

```typescript
{
  type: "title-bullets",
  badge: "카테고리명",
  title: "메인 타이틀",
  bullets: [
    "설명 항목 1",
    "설명 항목 2",
    "설명 항목 3",
  ],
  theme: "blue",
}
```

## 마크다운 패턴

```markdown
# [뱃지] 타이틀

- 불릿 항목 1
- 불릿 항목 2
- 불릿 항목 3
```

## 작성 원칙

- 한 슬라이드에 불릿은 4개 이하를 기본으로 본다
- 각 불릿은 한 문장으로 짧게 쓴다
- 첫 불릿이 가장 중요한 메시지여야 한다
- 이미지가 필요해지면 split 또는 title-image로 바꾼다

## 레이아웃 판단

- 기본 레이아웃은 1단 불릿 리스트다
- 하지만 불릿 수가 많거나, 3~4개 불릿이 모두 중간 길이 이상이면 에이전트가 2단 레이아웃으로 바꿔도 된다
- 2단 레이아웃의 목적은 정보를 더 많이 넣는 것이 아니라, 같은 밀도를 더 안정적으로 읽히게 만드는 것이다
- 2단으로 바꿔도 첫 불릿부터의 우선순위는 유지되어야 한다
- 불릿이 너무 길어서 2단에서도 답이 안 나오면 슬라이드를 둘로 나누는 편이 우선이다

## 2단 레이아웃 권장 상황

- 불릿이 4개이고 각 항목 길이가 비슷할 때
- 3개 불릿이지만 각 항목이 2줄 가까이 길어질 때
- 발표자가 한 화면에서 두 묶음으로 비교하거나 병렬 설명하고 싶을 때

## 2단 레이아웃 비권장 상황

- 불릿이 2개 이하일 때
- 첫 불릿만 압도적으로 중요하고 나머지가 보조일 때
- 메시지가 순차 흐름이라 위에서 아래로 읽혀야 할 때
