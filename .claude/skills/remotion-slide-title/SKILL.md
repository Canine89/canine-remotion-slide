# Title Slide

## 언제 쓰나

- 섹션 구분, 표지, 마무리 슬라이드처럼 제목 하나가 메시지의 전부일 때
- 이미지, 불릿, 태그 없이 타이포그래피만으로 임팩트를 줄 때
- 발표 흐름에서 "잠깐 멈춤"이나 "주제 전환"을 알릴 때

## 타입

```typescript
{
  type: "title",
  badge: "카테고리명",
  title: "메인 타이틀",
  subtitle: "보조 설명",  // optional
  theme: "dark",
}
```

## 마크다운 패턴

```markdown
# [뱃지] 타이틀
서브텍스트 (선택)
```

헤딩 아래에 불릿(`-`), 태그(`>`), 이미지(`![]()`), 추상 마커(`~`), evolution 마커(`<<`/`>>`) 없이 일반 텍스트만 있거나 아무것도 없으면 `title` 타입으로 감지된다.

## 작성 원칙

- 제목은 짧고 강하게, 한 줄에 담긴다
- 서브텍스트는 1줄 이내로 유지한다 — 길어지면 title-bullets로 전환
- 불릿이 필요하면 이 타입이 아니라 title-bullets를 쓴다
- 화면 중앙에 제목만 크게 나오므로, 제목 자체가 충분히 의미를 전달해야 한다

## title vs title-bullets 구분

| 상황 | 타입 |
|------|------|
| 제목만, 또는 제목 + 짧은 서브텍스트 | `title` |
| 제목 + 설명 항목 나열 | `title-bullets` |

마크다운에 `-` 불릿이 하나라도 있으면 `title-bullets`로 감지된다.
